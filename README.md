# Men Stack Session Auth
A full-stack app built to practice session-based authentication with Express and MongoDB.

## Description
This project was built to practice user authentication in a MEN stack application. It focuses on signing up, logging in, managing sessions, and protecting routes so only signed-in users can access certain parts of the app.

It was a useful way to improve my understanding of authentication, user data, and how secure access works in full-stack development.

## Planning
Before building, the main focus was understanding the login flow and how session-based authentication works.

Planning included:

- Planning protected routes
- Thinking about session storage
- Organising controllers, models, and middleware

## Brief
The goal of this lab was to practice session-based authentication in a MEN stack project.

The app needed to show how users can create accounts, log in, and access protected content securely.

## Build Process
The project started with setting up the Express server, MongoDB connection, and session middleware. After that, user models and authentication routes were added.

Once signup and login were working, middleware was used to protect routes and control access based on whether a user was signed in.
